## Что было сделано:

### 1. Динамические аватарки сотрудников

**Проблема:** В карточках сотрудников не было фотографий, что создавало визуальные "дыры" в интерфейсе.

**Решение:** Интеграция с сервисом UI Avatars для автоматической генерации аватарок с инициалами.

**Изменения в `frontend/src/views/Staff.vue`:**

#### Добавлен блок аватарки в шаблон:
```vue
<div class="staff-avatar">
  <img :src="getAvatarUrl(person)" :alt="person.full_name" />
</div>
```

#### Добавлена функция генерации URL аватарки:
```javascript
const getAvatarUrl = (person) => {
  // Если есть фото, используем его
  if (person.photo_url) {
    return person.photo_url
  }
  
  // Иначе генерируем аватарку с инициалами
  const name = encodeURIComponent(person.full_name)
  return `https://ui-avatars.com/api/?name=${name}&background=random&color=fff&size=200&bold=true&font-size=0.4`
}
```

#### Добавлены стили для аватарок:
```css
.staff-avatar {
  width: 120px;
  height: 120px;
  border-radius: 50%;
  overflow: hidden;
  margin-bottom: 1rem;
  border: 3px solid rgba(255, 255, 255, 0.4);
  box-shadow: 0 4px 20px rgba(0, 0, 0, 0.3);
  transition: all 0.3s;
}

.staff-card:hover .staff-avatar {
  transform: scale(1.05);
  border-color: rgba(255, 255, 255, 0.6);
}
```

**Преимущества:**
- Красивые круглые аватарки с инициалами вместо пустых мест
- Случайные цвета фона для каждого сотрудника
- Автоматическая поддержка кириллицы
- Плавная анимация при наведении
- Если в будущем добавятся реальные фото, они будут использоваться автоматически

**Параметры UI Avatars:**
- `name` - ФИО сотрудника (автоматически извлекаются инициалы)
- `background=random` - случайный цвет фона
- `color=fff` - белый цвет текста
- `size=200` - размер изображения 200x200px
- `bold=true` - жирный шрифт
- `font-size=0.4` - размер шрифта 40% от размера аватарки

### 2. Таймер бездействия для киоска

**Проблема:** В режиме информационного киоска пользователи могут уйти, оставив приложение на внутренней странице, что неудобно для следующих посетителей.

**Решение:** Автоматический возврат на главную страницу после 3 минут бездействия.

**Изменения в `frontend/src/App.vue`:**

#### Добавлен механизм отслеживания активности:
```javascript
import { useRouter } from 'vue-router'

const router = useRouter()
const INACTIVITY_TIMEOUT = 3 * 60 * 1000 // 3 минуты
let inactivityTimer = null

const resetInactivityTimer = () => {
  if (inactivityTimer) {
    clearTimeout(inactivityTimer)
  }

  inactivityTimer = setTimeout(() => {
    if (router.currentRoute.value.path !== '/') {
      console.log('Таймер бездействия: возврат на главную')
      router.push('/')
    }
  }, INACTIVITY_TIMEOUT)
}

const handleUserActivity = () => {
  resetInactivityTimer()
}
```

#### Добавлены слушатели событий:
```javascript
onMounted(() => {
  const events = ['mousedown', 'mousemove', 'keypress', 'scroll', 'touchstart', 'click']
  
  events.forEach(event => {
    document.addEventListener(event, handleUserActivity, true)
  })

  resetInactivityTimer()
})

onUnmounted(() => {
  if (inactivityTimer) {
    clearTimeout(inactivityTimer)
  }

  const events = ['mousedown', 'mousemove', 'keypress', 'scroll', 'touchstart', 'click']
  
  events.forEach(event => {
    document.removeEventListener(event, handleUserActivity, true)
  })
})
```

**Отслеживаемые события:**
- `mousedown` - нажатие кнопки мыши
- `mousemove` - движение мыши
- `keypress` - нажатие клавиши
- `scroll` - прокрутка страницы
- `touchstart` - касание экрана (для сенсорных киосков)
- `click` - клик мыши

**Логика работы:**
1. При любой активности пользователя таймер сбрасывается
2. Если активности нет 3 минуты, происходит автоматический переход на главную
3. Если пользователь уже на главной странице, переход не происходит
4. При размонтировании компонента все слушатели корректно удаляются

**Преимущества:**
- Киоск всегда готов к работе для нового посетителя
- Не мешает активным пользователям (таймер постоянно сбрасывается)
- Поддержка как мыши, так и сенсорных экранов
- Нет утечек памяти благодаря правильной очистке слушателей
- Логирование в консоль для отладки

## Тестирование:

### Проверка аватарок:
1. Откройте страницу "Сотрудники"
2. Убедитесь, что у каждого сотрудника есть круглая аватарка с инициалами
3. Проверьте, что цвета фона разные для разных сотрудников
4. Наведите курсор - аватарка должна увеличиться

### Проверка таймера бездействия:
1. Перейдите на любую страницу кроме главной (например, "Расписание")
2. Не трогайте мышь/клавиатуру 3 минуты
3. Приложение должно автоматически вернуться на главную
4. В консоли браузера появится сообщение: "Таймер бездействия: возврат на главную"

### Проверка сброса таймера:
1. Перейдите на страницу "Сотрудники"
2. Подождите 2 минуты
3. Пошевелите мышью или прокрутите страницу
4. Таймер должен сброситься, и вы останетесь на странице
