## Маршрут до кабинета (графовая навигация, вариант C)

**Цель:** на странице `/map?room=315` рисовать линию маршрута от главного
входа до целевого кабинета, в т.ч. с переходом между этажами.

> 📖 **Подробная инструкция, как разметить этажи (для пользователя):**
> [`docs/navigation-setup-guide.md`](navigation-setup-guide.md). Описывает
> весь процесс пошагово: от поиска координат в DevTools до проверки
> готового JSON.

### Что нужно подготовить

1. **Граф здания** — JSON вида:
   ```json
   {
     "nodes": [
       { "id": "f1-entrance", "floor": 1, "x": 410, "y": 830, "type": "entrance" },
       { "id": "f1-stairs-center", "floor": 1, "x": 410, "y": 540, "type": "stairs" },
       { "id": "f2-stairs-center", "floor": 2, "x": 410, "y": 540, "type": "stairs" },
       { "id": "room-214", "floor": 2, "x": 290, "y": 200, "type": "room", "room": "214" }
     ],
     "edges": [
       ["f1-entrance", "f1-stairs-center"],
       ["f1-stairs-center", "f2-stairs-center"]
     ]
   }
   ```
   Хранить в `frontend/public/map-graph.json`.

2. **Координаты узлов** — извлечь скриптом из `floor2.svg / floor3.svg /
   floor4.svg` (если в SVG есть `<text>` с номерами кабинетов — заготовка
   получается автоматически), плюс ручная разметка точек коридоров через
   режим `?edit=1` в `Map.vue` (клик копирует координаты).

3. **Inline-загрузка SVG этажей.** Сейчас 2–4 этажи — это `<img>`. Чтобы
   рисовать `<polyline>` поверх, надо загружать SVG inline (через `fetch +
   v-html` или превратить в Vue-компоненты). 1 этаж уже inline в
   `MapFloor2.vue`.

### Алгоритм

- A* (или Dijkstra) по графу, ~50 строк JS, без зависимостей.
- Вход: `startNode` (всегда главный вход) + `endNode` (узел кабинета).
- Выход: массив узлов в порядке прохождения.

### Отрисовка

- `<polyline points="x1,y1 x2,y2 ..." stroke="#2563EB" stroke-width="6"
  stroke-dasharray="14 8" fill="none" />` поверх SVG этажа.
- CSS-анимация `stroke-dashoffset` — линия «бежит» к цели.
- Старт — зелёный кружок («Вы здесь»), цель — пульсирующий красный.
- Если маршрут проходит несколько этажей — на текущем рисуется только
  часть, плюс плашка «↑ Поднимитесь на 3 этаж» с кнопкой переключения.

### Оценка трудозатрат

| Этап                                                    | Время |
|---------------------------------------------------------|-------|
| Inline-загрузка SVG на этажах 2–4                       | 30 мин |
| Скрипт извлечения позиций кабинетов из SVG              | 30 мин |
| Ручная разметка узлов коридоров и лестниц (4 этажа)     | 2–3 ч |
| A* + отрисовка polyline + анимация                      | 2 ч  |
| UX (плашка перехода между этажами, переключение этажей) | 1 ч  |
| **Итого**                                               | 6–8 ч |

### Подводные камни

- viewBox у разных SVG может отличаться → координаты узлов привязаны к
  конкретному этажу через поле `floor`.
- При ремонте/изменении планов придётся пересоставлять граф (но это
  данные на год вперёд).
- Для надёжного UX «вы здесь» нужно знать, где физически стоит стенд —
  это конфигурируется в `frontend/src/config.js` или env-переменной.

### Альтернативы (если C — слишком много)

- **Вариант A (~30 мин):** просто переключение этажа + подсветка
  кабинета. На 1 этаже подсвечивается реальный `<rect>`, на 2–4 — overlay
  кружок поверх картинки. Уже даёт пользу.
- **Вариант B (~2 ч):** A + текстовая инструкция «1. Вход → 2. Подняться
  на 3 этаж → 3. Правое крыло → 4. Кабинет 315» по таблице
  «кабинет → крыло/лестница».
