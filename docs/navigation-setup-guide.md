# Как разметить этажи для навигации (гайд для чайника)

Этот документ — **пошаговая инструкция**, как превратить наши плоские планы
этажей в работающую навигацию «от входа до кабинета». Если ты никогда не
работал с SVG, графами и JSON — ничего страшного, всё описано простым языком.

---

## 0. Что в итоге получится

Студент тапает по строчке расписания «Сети, ауд. 314» — карта сама
переключается на 3 этаж, на плане появляется **синяя пунктирная линия**,
которая ведёт от главного входа до двери кабинета. Если кабинет на другом
этаже — внизу появляется плашка «↑ Поднимитесь на 3 этаж», нажимаешь — карта
переключается, линия продолжается. Всё.

Чтобы это заработало, нужно один раз вручную «нарисовать» здание в виде
**графа** — точек и связей между ними. Это и есть наша задача.

---

## 1. Что нужно знать про граф (теория за 60 секунд)

**Граф** — это просто две вещи:

1. **Узлы (nodes)** — точки в пространстве. У каждой точки есть:
   - уникальный ID (например, `f3-corridor-center`)
   - этаж (1, 2, 3, 4)
   - координаты `x` и `y` на плане этажа
   - тип: вход / коридор / лестница / кабинет

2. **Рёбра (edges)** — связи между точками. Просто пара ID:
   `["f1-entrance", "f1-corridor-center"]` означает «можно пройти из главного
   входа в центр коридора 1 этажа».

Алгоритм поиска маршрута (A*) дальше сам разберётся. Наша работа — **только
расставить точки и соединить их правильно**.

## 3. Сначала — какие точки расставлять

На каждом этаже размечаем **четыре типа точек**:

### 3.1. Точки кабинетов (`type: "room"`)

Одна точка возле двери каждого кабинета. **Не в центре кабинета**, а именно
у двери, в коридоре. Это очень важно: маршрут заканчивается в коридоре
напротив двери, а не «внутри стены».

```
   ┌─────────────┐
   │             │
   │  Кабинет    │
   │    314      │
   │             │
   └────┬────────┘
        │
   • ── │ ── коридор   ← вот здесь ставим точку
        │
```

### 3.2. Точки коридоров (`type: "corridor"`)

Промежуточные точки **вдоль каждого коридора**, через каждые 5–8 метров (на
плане ~70–100 пикселей). Они нужны, чтобы маршрут шёл по коридору, а не «по
прямой через стены».

Главное правило: **в каждом повороте коридора должна быть точка**. Если
коридор идёт прямо — достаточно 1–2 точек на 10 метров.

### 3.3. Точки лестниц (`type: "stairs"`)

У каждой лестницы — **по одной точке на этаж, в одинаковом месте на плане**.
То есть если центральная лестница на 1 этаже стоит в координатах (410, 540),
то на 2, 3, 4 этажах она тоже должна быть в (410, 540) — это позволит
правильно «склеить» этажи.

### 3.4. Точки входов (`type: "entrance"`)

На 1 этаже — главный вход (и боковые, если ими пользуются). Это всегда
**стартовая точка** маршрута.

## 4. Как найти координаты точки

Тут самое муторное, но не сложное.

### Способ A. Через DevTools браузера (универсальный)

1. Запусти стенд локально (`npm run dev` в `frontend/`).
2. Открой http://localhost:5173/map в Chrome.
3. Переключись на нужный этаж.
4. Открой DevTools (F12) → вкладка Elements.
5. Нажми **«Pick element»** (стрелочка слева сверху в DevTools, либо
   Ctrl+Shift+C) и наведи курсор на нужное место плана.
6. Чтобы получить **точные координаты в системе SVG**, открой Console и
   введи:
   ```js
   document.querySelector('.svg-map-wrapper svg').addEventListener('click', (e) => {
     const svg = e.currentTarget;
     const pt = svg.createSVGPoint();
     pt.x = e.clientX;
     pt.y = e.clientY;
     const ctm = svg.getScreenCTM().inverse();
     const p = pt.matrixTransform(ctm);
     console.log(`x=${Math.round(p.x)}, y=${Math.round(p.y)}`);
   });
   ```
   Теперь каждый клик по плану печатает координаты в Console. Кликаешь —
   копируешь — пишешь в JSON.

### Способ B. Прямо в SVG-файле (для этажей 2–4)

Открой `frontend/public/floor2.svg` в текстовом редакторе. Это XML —
там видно `<text x="290" y="200">214</text>`. Это значит, что номер
кабинета 214 нарисован в точке (290, 200) — **примерно** такие же
координаты для точки кабинета и нужны (плюс-минус 20 пикселей в сторону
двери).

### Способ C. Линейкой на распечатке (для самых упорных)

Распечатай план, измерь линейкой расстояние в мм от левого края до точки
по X и от верхнего края до точки по Y, потом пересчитай через коэффициент
(SVG-ширина / ширина в мм). Долго и неудобно, но работает.

## 5. Формат JSON

Один файл на всё здание: `frontend/public/map-graph.json`. Структура:

```json
{
  "nodes": [
    { "id": "f1-entrance",          "floor": 1, "x": 410, "y": 830, "type": "entrance", "label": "Главный вход" },
    { "id": "f1-corridor-center",   "floor": 1, "x": 410, "y": 690, "type": "corridor" },
    { "id": "f1-stairs-center",     "floor": 1, "x": 410, "y": 540, "type": "stairs"   },
    { "id": "f2-stairs-center",     "floor": 2, "x": 410, "y": 540, "type": "stairs"   },
    { "id": "f2-corridor-left",     "floor": 2, "x": 290, "y": 540, "type": "corridor" },
    { "id": "f2-corridor-left-1",   "floor": 2, "x": 290, "y": 400, "type": "corridor" },
    { "id": "room-214",             "floor": 2, "x": 290, "y": 200, "type": "room", "room": "214" }
  ],
  "edges": [
    ["f1-entrance",        "f1-corridor-center"],
    ["f1-corridor-center", "f1-stairs-center"],
    ["f1-stairs-center",   "f2-stairs-center"],
    ["f2-stairs-center",   "f2-corridor-left"],
    ["f2-corridor-left",   "f2-corridor-left-1"],
    ["f2-corridor-left-1", "room-214"]
  ]
}
```

### Правила именования ID

Чтобы не запутаться, используй такую систему:

| Тип       | Шаблон                              | Пример                  |
|-----------|-------------------------------------|-------------------------|
| Вход      | `f{этаж}-entrance` или `-side`      | `f1-entrance`           |
| Коридор   | `f{этаж}-corridor-{секция}`         | `f3-corridor-right-2`   |
| Лестница  | `f{этаж}-stairs-{позиция}`          | `f2-stairs-center`      |
| Кабинет   | `room-{номер}`                      | `room-314`              |

### Правила рёбер

- Рёбра двусторонние — порядок ID в паре не важен.
- Не нужно прописывать ребро от лестницы к каждой точке: достаточно
  соединить её **с ближайшей точкой коридора**, а коридор уже соединён
  дальше.
- Лестничный «переход между этажами» — это **обычное ребро** между двумя
  точками-лестницами на разных этажах: `["f1-stairs-center",
  "f2-stairs-center"]`. Алгоритм поймёт.

## 6. Пошаговый план разметки одного этажа

Возьмём для примера **2 этаж**.

### Шаг 1. Открой план
Запусти стенд, открой `/map`, переключись на 2 этаж. Параллельно открой
`floor2.svg` в редакторе — оттуда удобно подсмотреть координаты номеров
кабинетов.

### Шаг 2. Точки лестниц
Найди на плане все лестницы. Обычно в коридорном здании их 2–3:
центральная и боковые. Поставь точку в **геометрическом центре**
лестничной площадки.

```json
{ "id": "f2-stairs-center", "floor": 2, "x": 410, "y": 540, "type": "stairs" },
{ "id": "f2-stairs-left",   "floor": 2, "x":  90, "y": 670, "type": "stairs" },
{ "id": "f2-stairs-right",  "floor": 2, "x": 720, "y": 670, "type": "stairs" }
```

**ВАЖНО:** запомни эти координаты — на остальных этажах используй
**точно такие же** для лестниц с тем же названием.

### Шаг 3. «Хребет» коридора
Пройдись по плану и расставь точки коридора **на каждом перекрёстке** и в
длинных прямых участках — каждые 100–150 пикселей.

Пример: длинный коридор тянется горизонтально от x=60 до x=760, y=690.
Размечаем:

```json
{ "id": "f2-corridor-1", "floor": 2, "x": 100, "y": 690, "type": "corridor" },
{ "id": "f2-corridor-2", "floor": 2, "x": 250, "y": 690, "type": "corridor" },
{ "id": "f2-corridor-3", "floor": 2, "x": 410, "y": 690, "type": "corridor" },
{ "id": "f2-corridor-4", "floor": 2, "x": 560, "y": 690, "type": "corridor" },
{ "id": "f2-corridor-5", "floor": 2, "x": 720, "y": 690, "type": "corridor" }
```

И связи:

```json
["f2-corridor-1", "f2-corridor-2"],
["f2-corridor-2", "f2-corridor-3"],
["f2-corridor-3", "f2-corridor-4"],
["f2-corridor-4", "f2-corridor-5"]
```

### Шаг 4. Подключи лестницы к коридору
Каждую лестничную точку соедини с **ближайшей точкой коридора**:

```json
["f2-stairs-left",   "f2-corridor-1"],
["f2-stairs-center", "f2-corridor-3"],
["f2-stairs-right",  "f2-corridor-5"]
```

### Шаг 5. Точки кабинетов и связи к коридору
Для каждого кабинета — точка у двери и ребро к ближайшей точке коридора.

```json
{ "id": "room-214", "floor": 2, "x": 290, "y": 540, "type": "room", "room": "214" },
{ "id": "room-215", "floor": 2, "x": 510, "y": 540, "type": "room", "room": "215" }
```

```json
["room-214", "f2-corridor-2"],
["room-215", "f2-corridor-4"]
```

### Шаг 6. Проверь, что всё связно
Простой тест: возьми любые два узла на этаже, мысленно пройди по
рёбрам — можно ли дойти? Если нет — значит, где-то не хватает ребра.

### Шаг 7. Повтори для каждого этажа
Этажи 1, 2, 3, 4 размечаются одинаково. Только на 1 этаже добавь точку
**главного входа** (`type: "entrance"`).

### Шаг 8. Соедини этажи между собой
Это всего несколько строк. Для каждой лестницы — ребро между её точками
на соседних этажах:

```json
["f1-stairs-center", "f2-stairs-center"],
["f2-stairs-center", "f3-stairs-center"],
["f3-stairs-center", "f4-stairs-center"],

["f1-stairs-left", "f2-stairs-left"],
["f2-stairs-left", "f3-stairs-left"],
["f3-stairs-left", "f4-stairs-left"]
```

## 8. Как проверить, что граф не сломан

После того как заполнишь файл:

1. **Проверка JSON-синтаксиса.** Открой файл в VS Code — он подчеркнёт
   красным любую опечатку (лишняя запятая, незакрытая скобка). Или проверь
   через https://jsonlint.com/.

2. **Проверка уникальности ID.** В консоли выполни:
   ```bash
   cd frontend/public
   node -e "
   const g = JSON.parse(require('fs').readFileSync('map-graph.json'));
   const ids = g.nodes.map(n => n.id);
   const dup = ids.filter((id, i) => ids.indexOf(id) !== i);
   console.log(dup.length ? 'Дубликаты: ' + dup.join(', ') : 'Все ID уникальны');
   "
   ```

3. **Проверка ссылок в рёбрах.** Каждый ID в `edges` должен присутствовать
   в `nodes`. Тоже проверяется парой строк JS — попроси меня (Devin)
   написать валидатор, я добавлю.

4. **Визуальная проверка.** Когда я доделаю фичу маршрутов, в режиме
   `?debug=1` карта будет показывать **все узлы цветными точками** и
   **рёбра — тонкими линиями**. Сразу видно, где не хватает связи или где
   точка съехала.

## 9. Частые ошибки

- ❌ **Точка кабинета внутри стены кабинета.** Поставь её в коридоре у двери.
- ❌ **Точка коридора посреди кабинета.** Это бывает когда в SVG номер
  кабинета нарисован в центре — не путай координаты текста с координатами
  двери.
- ❌ **Лестница на 1 этаже в (410, 540), а на 2 этаже в (415, 545).**
  Должны совпадать. Алгоритм этого не требует, но визуально маршрут
  будет «ломаться» при переходе.
- ❌ **Забыл соединить этажи.** Если нет рёбер между f1-stairs и
  f2-stairs — алгоритм не найдёт путь и выдаст «маршрут не найден».
- ❌ **Описал только половину рёбер.** В коридоре все соседние точки
  должны быть соединены.
- ❌ **Кабинет 314 как `"room": "314"` в одном месте и `"room-314"` в
  другом.** В поле `"room"` пишется только номер ("314"), в `"id"` —
  с префиксом ("room-314").

## 10. Что делает Devin после твоей разметки

Когда `map-graph.json` готов, я делаю всё остальное (ничего больше от
тебя не нужно):

2. Реализую A* поиск пути.
3. Превращаю этажи 2–4 из `<img>` в inline SVG (чтобы можно было рисовать
   `<polyline>` поверх).
4. Рендерю синюю пунктирную линию маршрута поверх плана с CSS-анимацией
   "бегущая дорожка".
5. Делаю плашку «↑ Поднимитесь на 3 этаж» для межэтажных маршрутов.
6. Подключаю кнопку «На карте» в расписании — она будет передавать
   `?room=314` в URL, карта сама найдёт цель.
7. Пишу автотесты (граф связный, все номера кабинетов из расписания
   присутствуют в графе и т.д.).

## 11. Сколько времени займёт у тебя

Если делаешь без перерывов:

| Этап                             | Время       |
|----------------------------------|-------------|
| Освоиться с JSON и DevTools      | 15–30 мин   |
| 1 этаж (вход, коридоры, лестницы)| 30–45 мин   |
| Каждый из этажей 2, 3, 4         | по 30 мин   |
| Проверка, исправление опечаток   | 15–30 мин   |
| **Итого**                        | **2.5–3 ч** |

Можно разбить на два захода: за один присест разметить 1–2 этажа, потом
доделать остальные. **Разбивать на много заходов не стоит — забудешь
свою же систему именования.**
