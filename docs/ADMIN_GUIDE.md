# Руководство администратора

## Доступ к панели управления

1. Откройте в браузере: `http://<адрес-стенда>/admin/login`
2. При первом запуске система предложит создать учётную запись администратора (логин + пароль)
3. После создания — произойдёт автоматический вход
4. При последующих входах — введите логин и пароль

> **Важно**: после создания первого администратора регистрация новых недоступна через веб-интерфейс. Для сброса пароля удалите таблицу `admins` из БД и перезапустите сервер.

## Вкладки панели

### 📰 Новости

| Действие | Описание |
|----------|----------|
| Список | Все новости с датой публикации и статусом активности |
| Добавить | Кнопка «+ Добавить» → форма с заголовком, описанием, иконкой, ссылкой |
| Редактировать | Кнопка ✏️ рядом с записью |
| Удалить | Кнопка 🗑️ (требует подтверждения) |
| Скрыть/Показать | Измените поле «Активна» при редактировании |

**Поля новости:**
- **Заголовок** — отображается на стенде (обязательное)
- **Описание** — краткий текст
- **Иконка** — emoji-иконка (по умолчанию 📰)
- **Ссылка на источник** — URL оригинальной новости
- **Активна** — если «Нет», новость не отображается на стенде

> Новости также автоматически парсятся с сайта kraskrit.ru каждые 6 часов.

### 👥 Сотрудники

| Действие | Описание |
|----------|----------|
| Список | Все сотрудники с ФИО, должностью, отделом, кабинетом |
| Добавить | Кнопка «+ Добавить» → форма |
| Редактировать | Кнопка ✏️ |
| Удалить | Кнопка 🗑️ |

**Поля сотрудника:**

> Сотрудники также загружаются массово через скрипт `load_staff.py`.

| Действие | Описание |

### 📊 Аналитика

Отображает:
- **Всего посещений** — суммарное количество запросов к API
- **Сегодня** — посещения за текущий день
- **За неделю** — посещения за последние 7 дней
- **Популярные запросы расписания** — какие группы/аудитории ищут чаще всего
- **Топ страниц** — самые запрашиваемые эндпоинты API

> Данные обновляются в реальном времени при каждом запросе к API.

## Управление расписанием

Расписание загружается автоматически из Яндекс.Диска. Ручные способы:

### Импорт из файла

```powershell
cd backend
.\venv\Scripts\python.exe ..\load_schedule.py путь/к/файлу.xlsx
```

Поддерживаются форматы `.xlsx` и `.pdf`.

### Синхронизация с Яндекс.Диском

Настройте в `backend/.env`:
```
YANDEX_DISK_PUBLIC_URL=https://disk.yandex.ru/d/ваша-папка
YANDEX_DISK_SYNC_INTERVAL_HOURS=3
```

Синхронизация запускается автоматически при старте сервера и повторяется каждые N часов.

### Очистка мусорных групп

```powershell
cd backend
.\venv\Scripts\python.exe ..\scripts\cleanup_garbage_groups.py --apply
```

Флаг `--apply` — реальное удаление. Без него — режим dry-run (только показ).

## Запуск и перезапуск сервера

### Запуск backend

```powershell
cd backend
.\venv\Scripts\Activate.ps1
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### Запуск frontend (разработка)

```powershell
cd frontend
npm run dev
```

### Сборка frontend (production)

```powershell
cd frontend
npm run build
```

Результат — в папке `frontend/dist/`.

## Переменные окружения (backend/.env)

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `DATABASE_URL` | `sqlite:///./kkrit.db` | Путь к базе данных |
| `SECRET_KEY` | `your-secret-key...` | Ключ для JWT-токенов (**измените в production!**) |
| `DEBUG` | `True` | Режим отладки |
| `CORS_ORIGINS` | `http://localhost:5173` | Разрешённые origins (через запятую) |
| `RATE_LIMIT_ENABLED` | `True` | Включить ограничение запросов |
| `YANDEX_DISK_PUBLIC_URL` | — | URL публичной папки Яндекс.Диска |
| `YANDEX_DISK_SYNC_INTERVAL_HOURS` | `3` | Интервал синхронизации (часы) |
| `YANDEX_DISK_RECURSIVE` | `True` | Обходить подпапки на Яндекс.Диске |

## Устранение неисправностей

| Проблема | Решение |
|----------|---------|
| «Database is locked» | Убедитесь, что `SQLITE_WAL_MODE=True` в .env; перезапустите сервер |
| Расписание не обновляется | Проверьте `YANDEX_DISK_PUBLIC_URL` в .env; посмотрите логи сервера |
| Новости не парсятся | Проверьте доступность kraskrit.ru; посмотрите логи |
| Не могу войти в админку | Удалите таблицу `admins` из БД и перезапустите сервер для повторной регистрации |
| 401 Unauthorized | Токен истёк (8 часов) — перезайдите в админку |
