# Загрузка расписания

Расписание пар в ККРИТ живёт в трёх местах:

1. **Расписание звонков** — статичное, хранится в коде
   (<code>backend/app/core/bell_schedule.py</code>). Это единственный
   источник правды о том, когда какая пара начинается и заканчивается.
   Понедельник отличается от остальных дней недели (линейка + классный
   час), это учтено.
2. **Конкретное расписание на дату** (какая группа где учится) — хранится
   в таблице <code>schedules</code> в БД. Импортируется из Excel / PDF.
3. **API** — `/api/v1/schedule/bells` отдаёт расписание звонков,
   `/api/v1/schedule/group/{name}?date=YYYY-MM-DD` — конкретные пары на
   дату.

## Ручной импорт

```bash
# Excel
python load_schedule.py 13.04.2026.xlsx --date 2026-04-13

# PDF
python load_schedule.py 13.04.2026.pdf --date 2026-04-13
```

Повторный импорт идемпотентен — старые записи на ту же дату удаляются
перед вставкой.

## Автоматическая синхронизация с Яндекс.Диска

Файлы расписания публикуются на Яндекс.Диск в публичную папку. Бэкенд
умеет периодически опрашивать эту папку, скачивать новые PDF/Excel и
импортировать их в БД — **без OAuth-токена**, только по публичной ссылке.

### Включить

Задай переменные окружения:

```bash
# Обязательно: публичная ссылка на папку Яндекс.Диска. Можно указать как
# корень публичной папки, так и конкретный подкаталог — URL с подпапкой
# автоматически распарсится (API Яндекс.Диска не принимает такой URL
# напрямую, модуль разбивает его на public_key + path).
export YANDEX_DISK_PUBLIC_URL="https://disk.yandex.ru/d/XXXXXXXXXXXX/корпус пр-т ... 156"

# Опционально: куда сохранять скачанные файлы (по умолчанию data/schedule-downloads).
export SCHEDULE_DOWNLOAD_DIR="data/schedule-downloads"

# Опционально: интервал синхронизации в часах (по умолчанию 6).
export YANDEX_DISK_SYNC_INTERVAL_HOURS="6"
```

После этого при запуске бэкенда (`python run.py`) в логах появится:

```
📥 Синхронизация расписания с Яндекс.Диска: https://disk.yandex.ru/d/...
```

Первая синхронизация — через минуту после старта, далее раз в
`YANDEX_DISK_SYNC_INTERVAL_HOURS` часов.

### Ручной запуск

Можно дернуть синхронизацию из CLI в любой момент:

```bash
# Показать список файлов в папке (ничего не скачивая)
python -m app.services.yandex_disk \
  --url "https://disk.yandex.ru/d/XXXXX" --list-only

# Скачать + импортировать всё, у чего распознали дату в имени
python -m app.services.yandex_disk \
  --url "https://disk.yandex.ru/d/XXXXX" \
  --dest data/schedule-downloads

# Обход всех вложенных папок (архивы по месяцам и т.п.)
python -m app.services.yandex_disk \
  --url "https://disk.yandex.ru/d/XXXXX" --list-only --recursive
```

### Как распознаётся дата

Имя файла должно содержать дату в одном из форматов:

- `13.04.2026` (с точками)
- `13-04-2026` (с дефисами)
- `13_04_2026` (с подчёркиваниями)

Примеры, которые подойдут:

- `13.04.2026.pdf`
- `raspisanie-13-04-2026.xlsx`
- `schedule_13_04_2026_final.pdf`

Файлы без распознанной даты пропускаются (логируются, но не
импортируются).

### Альтернатива: приватный OAuth-токен

Если папка не публичная, можно перевести синхронизацию на
[WebDAV Яндекс.Диска](https://yandex.ru/dev/disk-api/doc/ru/reference/webdav)
или [REST API с OAuth-токеном](https://yandex.ru/dev/disk-api/doc/ru/reference/auth).
Текущая реализация специально использует публичный API без токенов, чтобы
не хранить секреты на сервере киоска.
