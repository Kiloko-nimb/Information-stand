# Парсинг новостей с сайта ККРИТ - Инструкция

## Что было реализовано

Система автоматического парсинга новостей с официального сайта колледжа https://kraskrit.ru/

### Backend компоненты:
- ✅ Модель `News` для хранения новостей в БД
- ✅ API endpoints `/api/v1/news` для получения новостей
- ✅ Сервис парсинга `news_parser.py` для загрузки с сайта
- ✅ Планировщик задач для автоматического обновления каждые 6 часов
- ✅ Таблица `news` в базе данных

### Frontend компоненты:
- ✅ Сервис `newsService.js` для работы с API
- ✅ Динамическая загрузка новостей в `Home.vue`
- ✅ Offline-кэширование в localStorage
- ✅ Обработка состояний: loading, error, empty

## Установка зависимостей

### Backend:
```bash
cd backend
pip install beautifulsoup4 requests lxml
```

**Примечание:** APScheduler может потребовать установки вручную из-за проблем с прокси:
```bash
pip install APScheduler==3.10.4
```

### Frontend:
Все зависимости уже установлены (используется axios).

## Запуск

### 1. Создание таблицы (уже выполнено):
```bash
python create_news_table.py
```

### 2. Запуск backend:
```bash
cd backend
python run.py
```

Backend автоматически:
- Запустит планировщик обновления новостей
- Сделает первую загрузку новостей при старте
- Будет обновлять новости каждые 6 часов

### 3. Запуск frontend:
```bash
cd frontend
npm run dev
```

## Настройка парсера

Файл `backend/app/services/news_parser.py` содержит логику парсинга.

**ВАЖНО:** Селекторы HTML нужно адаптировать под реальную структуру сайта kraskrit.ru:

```python
# Строка 48-49: Найти правильные CSS-классы для блоков новостей
news_blocks = soup.find_all('div', class_='news-item')  # Изменить класс!

# Строка 67-68: Найти правильные теги для заголовков

# Строка 71-72: Найти правильные классы для описания

```

### Как узнать правильные селекторы:

1. Открыть https://kraskrit.ru/news/ в браузере
2. Нажать F12 (DevTools)
3. Найти блок с новостью
4. Посмотреть его класс и структуру
5. Обновить селекторы в `news_parser.py`

## Изменение интервала обновления

В файле `backend/app/main.py` строка 42:
```python
start_scheduler(interval_hours=6)  # Изменить на нужное значение
```

Для тестирования можно поставить меньший интервал (например, 0.1 часа = 6 минут).

## Offline-режим

Frontend автоматически кэширует новости в localStorage:
- Кэш действителен 24 часа
- При отсутствии связи загружаются из кэша
- Если кэш пустой, показываются заглушки

## Troubleshooting

### Новости не загружаются:
1. Проверить, что backend запущен: `curl http://localhost:8000/health`
2. Проверить логи backend на наличие ошибок парсинга
3. Проверить, что сайт kraskrit.ru доступен
4. Проверить селекторы в `news_parser.py`

### Парсер не находит новости:
1. Запустить парсер вручную для отладки:
```bash
cd backend
python -m app.services.news_parser
```
2. Проверить HTML-структуру сайта
3. Обновить селекторы

### APScheduler не установлен:
Если есть проблемы с установкой через pip, можно:
1. Скачать wheel файл вручную
2. Установить из локального файла
3. Или временно отключить планировщик (закомментировать строки 42-43 в main.py)
