## 2. Ключевые технические решения

### 2.1. Отключение pointer-events для текста

**Проблема:** Текст номеров кабинетов блокировал клики по прямоугольникам.

**Решение:**
```css
text {
  pointer-events: none;
  user-select: none;
}
```

Это **критически важное** правило! Без него текст перехватывает события мыши и кабинеты не кликаются.

### 2.2. Интерактивные прямоугольники

Каждый кабинет представлен как `<rect>` с:
- Классом `.room` для базовых стилей
- Обработчиком `@click="selectRoom('214')"`
- Реактивным классом `:class="{ 'active': currentRoom === '214' }"`

**Пример:**
```vue
<rect x="220" y="140" width="145" height="110" 
      class="room" 
      :class="{ 'active': currentRoom === '214' }"
      @click="selectRoom('214')"
      fill="rgba(102, 126, 234, 0.1)" 
      stroke="transparent" 
      stroke-width="3"/>
```

### 2.3. Реактивное состояние

```javascript
const currentRoom = ref(null)

const selectRoom = (roomNumber) => {
  currentRoom.value = roomNumber
  console.log(`Выбран кабинет: ${roomNumber}`)

```

## 4. Визуальные эффекты

### Состояние по умолчанию:
```css
.room {
  cursor: pointer;
  fill: rgba(102, 126, 234, 0.1);
  transition: all 0.3s ease;
}
```

### При наведении (hover):
```css
.room:hover {
  fill: rgba(102, 126, 234, 0.3) !important;
  stroke: rgba(102, 126, 234, 0.8) !important;
  stroke-width: 4 !important;
}
```

### При выборе (active):
```css
.room.active {
  fill: rgba(102, 126, 234, 0.5) !important;
  stroke: rgba(102, 126, 234, 1) !important;
  stroke-width: 5 !important;
  filter: drop-shadow(0 0 10px rgba(102, 126, 234, 0.8));
}
```

**Эффекты:**
- Изменение прозрачности заливки
- Появление/усиление обводки
- Свечение вокруг выбранного кабинета
- Плавные переходы (0.3s)

## 6. Расширение функционала

### 6.1. Добавление информации о кабинете

Можно расширить функцию `selectRoom`:

```javascript
const selectRoom = (roomNumber) => {
  currentRoom.value = roomNumber
  
  // Получить информацию о кабинете из API
  fetchRoomInfo(roomNumber)
  
  // Показать модальное окно с деталями
  showRoomDetails(roomNumber)
}
```

### 6.2. Интеграция с расписанием

```javascript
const selectRoom = async (roomNumber) => {
  currentRoom.value = roomNumber
  
  // Получить расписание для этой аудитории
  const schedule = await api.get(`/schedule/room/${roomNumber}`)
  
  // Показать, какие занятия проходят в этом кабинете
  displaySchedule(schedule)
}
```

### 6.3. Поиск кабинета

```javascript
const searchRoom = (roomNumber) => {
  // Автоматически выделить кабинет
  currentRoom.value = roomNumber
  
  // Прокрутить к карте
  scrollToMap()
  
  // Показать подсказку
  showTooltip(`Кабинет ${roomNumber} выделен на карте`)
}
```

## 7. Адаптация под другие этажи

Чтобы создать карты для других этажей:

1. **Скопируйте компонент:**
   ```bash
   cp MapFloor2.vue MapFloor1.vue
   cp MapFloor2.vue MapFloor3.vue
   ```

2. **Замените SVG-код** на карту нужного этажа

3. **Обновите номера кабинетов** в:
   - Атрибутах `@click="selectRoom('XXX')"`
   - Условиях `:class="{ 'active': currentRoom === 'XXX' }"`
   - Текстовых элементах `<text>`

4. **Настройте координаты** прямоугольников под новую карту

## 11. Известные ограничения

1. **Номера кабинетов захардкожены** - можно вынести в конфигурацию
2. **Координаты прямоугольников подобраны вручную** - требуют точной настройки
3. **Нет автоматического определения кабинетов из SVG** - нужно добавлять вручную
4. **Одна карта на компонент** - для других этажей нужны отдельные компоненты
