## 🎯 О проекте

**ККРИТ Информационный стенд** — это полнофункциональное решение для цифровизации информационного пространства колледжа. Проект заменяет традиционные бумажные объявления и статические стенды на современный интерактивный сенсорный интерфейс, который:

- **Автоматически обновляется** — расписание и новости подгружаются без вмешательства человека
- **Адаптирован для сенсорных экранов** — крупные элементы управления, интуитивная навигация
- **Работает в режиме киоска** — защита от случайных действий, скринсейвер при простое
- **Предоставляет аналитику** — отслеживание популярных запросов для улучшения сервиса

### Целевая аудитория

| Группа пользователей | Что они получают |
|---------------------|------------------|
| **Абитуриенты и гости** | Быстрый поиск кабинетов, информация о приёмной комиссии |
| **Студенты** | Актуальное расписание пар, информация о преподавателях |
| **Преподаватели** | Расписание групп, кабинетов, контакты коллег |
| **Администрация** | Инструмент управления контентом, аналитика использования |

## ✨ Основные возможности

### 🔍 Поиск и навигация

- **Расписание занятий**
  - Поиск по группе, преподавателю или кабинету
  - Отображение текущей пары с таймером
  - Индикация статуса дня (учебный/выходной)
  - Расписание звонков с визуальным таймером

- **Справочник сотрудников**
  - Фильтрация по ФИО, должности, кафедре
  - Контактная информация и часы приёма
  - Фотографии (при наличии)

- **Карта здания**
  - Интерактивные планы этажей в формате SVG
  - Построение маршрутов от входа до кабинета
  - Информация о доступности для маломобильных групп

### 📰 Информационные блоки

- **Новости колледжа**
  - Автоматический парсинг с официального сайта
  - Ручное добавление через админ-панель
  - Категоризация и приоритеты отображения

- **Виджеты главной страницы**
  - Текущее время и дата
  - Статус учебного дня
  - Ближайшая пара
  - Анонсы мероприятий

### 🛡️ Режим киоска

- **Защита от простоя**
  - Скринсейвер после N минут бездействия
  - Предупреждение перед возвратом на главную
  - Автоматический сброс поиска

- **Оптимизация для 4K**
  - Масштабирование интерфейса под большие экраны
  - Чёткие шрифты и контрастные цвета
  - Адаптивная вёрстка

## 🏗️ Архитектура

Проект построен по принципу **клиент-сервер** с чётким разделением ответственности:

```
┌─────────────────────────────────────────────────────────────┐
│                     Сенсорный экран (Киоск)                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │              Frontend (Vue 3 + Vite)                  │  │
│  │  • Роутинг между страницами                           │  │
│  │  • Управление состоянием (Pinia)                      │  │
│  │  • HTTP-запросы к API                                 │  │
│  │  • Локальные хуки (useClock, useScreensaver)          │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                            ↕ HTTP/REST API
┌─────────────────────────────────────────────────────────────┐
│                   Backend (FastAPI + SQLite)                 │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │   API Layer │  │   Services  │  │   Data Sources      │  │
│  │  • Auth     │  │  • Yandex   │  │  • Яндекс.Диск      │  │
│  │  • Schedule │  │  • Parser   │  │  • Сайт колледжа    │  │
│  │  • Staff    │  │  • Import   │  │  • DOCX/XLSX/PDF    │  │
│  │  • News     │  │             │  │                     │  │
│  │  • Analytics│  │             │  │                     │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │              Database (SQLite + SQLAlchemy)           │  │
│  │  Tables: schedule, staff, rooms, news, admin, analytics│ │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### Фоновые задачи

Backend автоматически выполняет следующие задачи:

| Задача | Периодичность | Описание |
|--------|---------------|----------|
| Импорт расписания | Каждые 3 часа | Скачивание XLSX/PDF с Яндекс.Диска, парсинг, обновление БД |
| Парсинг новостей | Каждые 6 часов | Сбор публикаций с сайта колледжа |
| Очистка аналитики | Ежедневно | Удаление устаревших записей (> 30 дней) |
| Проверка целостности | При старте | Валидация таблиц, создание отсутствующих колонок |

## 📊 Источники данных

Проект агрегирует информацию из нескольких источников:

### 1. Расписание занятий

| Параметр | Значение |
|----------|----------|
| **Источник** | Публичная папка Яндекс.Диска |
| **Форматы** | XLSX (Excel), PDF |
| **Частота обновления** | Автоматически каждые 3 часа |
| **Ручной импорт** | `python load_schedule.py path/to/file.xlsx --date YYYY-MM-DD` |
| **Хранение** | `backend/data/schedule-downloads/` |
| **Манифест** | `.imported.json` (отслеживает обработанные файлы) |

**Процесс импорта:**
1. Скачивание файла из папки Яндекс.Диска
2. Определение формата (XLSX/PDF)
3. Парсинг структуры документа
4. Нормализация названий групп и кабинетов
5. Сохранение в таблицу `schedule`
6. Обновление справочника `rooms`

### 2. Новости колледжа

| Параметр | Значение |
|----------|----------|
| **Источник** | Официальный сайт колледжа (HTML) |
| **Парсер** | `services/news_parser.py` (BeautifulSoup4) |
| **Частота обновления** | Автоматически каждые 6 часов |
| **Поля** | Заголовок, текст, дата, ссылка, изображение |

### 3. Сотрудники (педсостав)

| Параметр | Значение |
|----------|----------|
| **Источник** | DOCX файл с сайта колледжа |
| **Импорт** | Ручной: `python load_staff.py` |
| **Обновление** | Через админ-панель или повторным запуском скрипта |
| **Поля** | ФИО, должность, кафедра, телефон, email, часы приёма |

### 4. Кабинеты

| Параметр | Значение |
|----------|----------|
| **Источник** | Автоматически из расписания (уникальные `room_number`) |
| **Ручное добавление** | Через админ-панель |
| **Дополнительные поля** | Этаж, корпус, описание, доступность для МГН |

### 5. Администраторы

## 🔧 Устранение неполадок

### Проблема: БД повреждена после сбоя

**Симптомы:** Ошибки SQLite, приложение не запускается

**Решение:**
```bash
cd backend
rm kkrit.db kkrit.db-shm kkrit.db-wal
# Перезапустите uvicorn — таблицы создадутся заново
```

### Проблема: Не загружается расписание

**Симптомы:** Пустой список групп/преподавателей

**Диагностика:**
1. Проверьте `YANDEX_DISK_PUBLIC_URL` в `.env`
2. Посмотрите логи uvicorn на ошибки парсинга
3. Проверьте наличие файлов в `data/schedule-downloads/`

**Решение:**
```bash
# Ручной импорт
python load_schedule.py path/to/manual_schedule.xlsx --date 2024-09-01
```

### Проблема: CORS ошибки в браузере

**Симптомы:** Frontend не может подключиться к API

**Решение:**
1. В `backend/.env` добавьте:
   ```ini
   CORS_ORIGINS=["http://localhost:5173","http://localhost:3000"]
   ```
2. Перезапустите backend

### Проблема: Киоск не уходит в скринсейвер

**Симптомы:** Экран не гаснет после простоя

**Решение:**
1. Проверьте `VITE_KIOSK_TIMEOUT_SEC` в `frontend/.env`
2. Убедитесь, что composable `useScreensaver` подключён в `App.vue`
3. Проверьте консоль браузера на ошибки JavaScript

### Проблема: Кодировка .env на Windows

**Симптомы:** Ошибки при чтении переменных окружения

**Решение:**
1. Откройте `.env` в Notepad++
2. Конвертируйте в UTF-8 без BOM
3. Сохраните и перезапустите приложение

## ❓ Часто задаваемые вопросы

### Q: Как добавить своё расписание?

**A:** Используйте один из способов:
1. Положите XLSX/PDF файл в папку Яндекс.Диска (если настроена автосинхронизация)
2. Запустите `python load_schedule.py file.xlsx --date YYYY-MM-DD`
3. Через админ-панель (в разработке)

### Q: Можно ли использовать PostgreSQL вместо SQLite?

**A:** Да. Измените `DATABASE_URL` в `.env`:
```ini
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/kkrit
```
Установите дополнительный пакет: `pip install asyncpg`

### Q: Как изменить время показа скринсейвера?

**A:** Отредактируйте `VITE_KIOSK_TIMEOUT_SEC` в `frontend/.env` (значение в секундах).

### Q: Где хранятся загруженные файлы расписания?

**A:** В директории `backend/data/schedule-downloads/`. Манифест обработанных файлов — `.imported.json`.

### Q: Как посмотреть статистику использования?

**A:** Откройте админ-панель → вкладка "Аналитика". Там отображаются:
- Топ поисковых запросов
- Количество запросов по времени суток
- Популярные разделы

### Q: Можно ли отключить сбор аналитики?

**A:** Да. Добавьте в `backend/.env`:
```ini
ANALYTICS_ENABLED=False
```
